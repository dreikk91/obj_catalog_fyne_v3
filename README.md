# Obj Catalog (Fyne) — АРМ Пожежної Безпеки 

Опис: це графічний десктоп-застосунок (автоматизоване робоче місце оператора), написаний на Go з використанням [Fyne](https://fyne.io/) для UI. Проєкт призначений для моніторингу станів об'єктів, перегляду та обробки подій і тривог.

## 🏗 Архітектура проекту (MVVM)

Проект використовує **MVVM (Model-View-ViewModel)** патерн та принципи «чистої архітектури» для чіткого розділення відповідальності між візуальним інтерфейсом, логікою відображення та джерелами даних.

### Структура репозиторію

* **`main.go`** — точка входу, ініціалізація логера, конфігурації та Fyne-додатка.
* **`pkg/application`** — оркестрація додатку, життєвий цикл та збірка UI.
* **`pkg/ui/`** — графічний інтерфейс: панелі (`alarm_panel.go`, `work_area.go` тощо), діалогові вікна. GUI **не має** прямого доступу до бази даних.
* **`pkg/ui/viewmodels/`** — шар ViewModel, який відповідає за підготовку даних для UI, обробку команд та синхронізацію стану між представленням і моделлю.
* **`pkg/contracts/`** — інтерфейси (`DataProvider`, `AdminProvider`) та DTO, через які взаємодіють різні шари архітектури.
* **`pkg/backend/`** — створює та налаштовує провайдери даних, повертаючи інтерфейси для використання у графічному шарі.
* **`pkg/data/`** — імплементації доступу до даних:
  * `db_provider.go` / `db_admin_provider.go` — робота з основною базою Firebird.
  * `casl_*.go` — модульна реалізація клієнта **CASL Cloud** (події, об'єкти, зони, протоколи тощо).
  * `phoenix_provider.go` — інтеграція із системою **Phoenix**.
* **`pkg/models/`** — доменні моделі сутностей системи.
* **`pkg/config/`** — конфігурація (робота з `fyne.Preferences`, збереження налаштувань БД).
* **`pkg/logger/`** — налаштована система логування (`zerolog` + `lumberjack`) із ротацією та детальними етапами запуску.

## 🔌 Робота з даними та багатоджерельність

Програма підтримує одночасну роботу з декількома системами (multi-source mode).
* **Firebird (головна БД):** Використовує драйвер `github.com/nakagami/firebirdsql` і обгортку `sqlx`. Реалізовано фоновий моніторинг стану (Health Check), що забезпечує автоматичне відновлення після втрати зв'язку.
* **CASL Cloud API:** Глибока інтеграція (отримання об'єктів, трансляція подій, класифікація тривог, обробка номерів приладів). Реалізацію розбито на дрібні модулі для підтримання чистоти коду.
* **Київстар / Vodafone:** Інтеграції для опитування стану SIM-карт в приладах.

## ⚙️ Збірка та запуск

### Режим розробки

1. Переконайтесь, що встановлено **Go 1.24+** (згідно з `go.mod`).
2. Завантажте залежності та запустіть:
   ```powershell
   go mod tidy
   go run main.go
   ```

### Пакування у виконуваний файл

Проєкт має систему скриптів для збірки з авто-підстановкою версії з Git (commit/time/dirty) та можливістю стиснення (UPX):

* **Звичайна Windows-збірка:**
  ```powershell
  .\scripts\build_app.ps1 -Output obj_catalog_fyne_v3.exe
  ```
* **Fyne Package (Standalone, з іконкою):**
  ```powershell
  .\scripts\package_fyne.ps1 -InstallFyne -TargetOS windows -Output obj_catalog_fyne_v3.exe -UseUPX
  ```

Перевірити версію зібраного бінарника:
```powershell
.\obj_catalog_fyne_v3.exe --version
```

## 📝 Логування

В системі запроваджено детальне логування (див. `LOGGING_SUMMARY.md` та `LOGGING_QUICK_REFERENCE.md`). 
* Логи пишуться графічно в консоль та зберігаються у папку `log/app.log` з архівацією за розміром.
* Окремо формується файл помилок `log/error.log`.
* Всі сеанси містять банер ініціалізації з детальним треком: ініціалізація → підключення до БД → моніторинг → UI.

## 🎨 Інші особливості
* **Теми:** Підтримка системної та налаштовуваної (світлої/темної) теми з перемикачем у програмі.
* **DPI:** Автоматичне налаштування під різні масштаби екрану (Fyne-особливість).
* **Тестування:** Велике покриття юніт-тестами (viewmodels, casl api, form states). Для локального тестування UI можна використовувати Mock Data Provider в `pkg/data`.
